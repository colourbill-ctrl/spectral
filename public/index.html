User guide

This app reads spectral measurements (CGATS, CSV, or CxF/X-3) and converts them to CIE XYZ + L*a*b* using iccDEV's IccColorimetry compiled to WebAssembly. All colour math — spectral→XYZ and XYZ→Lab — is performed by iccDEV; JavaScript only parses input and formats output, so the app exercises iccDEV's canonical math end to end.

Setup controls

Observer
The CIE standard colour-matching functions: 1931 (2°) — the graphic-arts and ICC default — or 1964 (10°), common in paper and display work.
Illuminant
The reference light the reflectance is viewed under. The available list depends on the Method: the calculator methods offer the built-in SPDs D50, D65, D93, A; the registry table offers the published tables D50, D65, A, LED-B1, F11. LED-B1 and F11 exist only as registry tables (they have no built-in SPD).
Method
The spectral→XYZ reduction algorithm:
  • Direct summation — naive rectangular sum at the measurement interval (the existing iccMAX baseline; least accurate on coarse data).
  • Weighting function — self-computed ASTM-style weights (WP56). A good general choice when data is not on a 1 or 5 nm grid.
  • Sprague → 1 nm — CIE 167:2005 Sprague reconstruction to 1 nm before summation (highest fidelity for smooth reflectance).
  • Registry LWL table (10 nm) — applies the ICC colorimetry-data registry's published least-squares ("LWL", Li et al. 2016 / ISO 13655) weighting table directly. Fixed grid 380–780 nm @ 10 nm (41 bands); your data must already be on that grid — it does not resample. This is the registry's authoritative result.
Interpolation
How spectra are resampled onto the integration grid (Linear / Cubic / Sprague). Disabled for the registry table, which maps band-for-band and has nothing to interpolate.
End handling
How a spectrum is extended past its measured ends when the integration grid is wider: Hold repeats the end value (ICC-recommended); Linear extrapolates from the last two samples. Disabled for the registry table.
Reflectance scale
Input units only: Auto-detect (percent if any value > 2, else fraction), 0–1, or 0–100. This rescales the input before conversion; it does not clip sample values — negative or > 1 samples pass through unchanged.

ICC-recommended settings

Per ICC TN-06-2025 (a local copy is bundled at docs/ICC_TN-06-2025.pdf):

For typical 10 nm reflectance data spanning 380–780 nm — the most authoritative result:
  • Method = Registry LWL table (10 nm) — TN-06 §5 calls the registry's least-squares weighting functions the state of the art.
  • Observer = 1931 (2°) — for ICC.1 / graphic-arts compatibility.
  • Illuminant = D50 — the ICC.1 PCS illuminant. With spectra in hand, use D50 directly rather than chromatically adapting from another illuminant.
If your data is not on the 380–780 / 10 nm grid (e.g. 5 nm, or 380–730 nm):
  • Method = Weighting (or Sprague → 1 nm) — TN-06 §4: use a weighting function for non-1/5 nm intervals; otherwise interpolate to 1 nm with Sprague.
  • End handling = Hold — TN-06 §3: extend the measured data to the function range by holding the last known value (not linear, not higher-order).
  • Interpolation = Sprague when using the Sprague method.

Caveats: the 10 nm registry tables are provisional (pending CIE TC1-101) and carry the registry's Y = 100 normalization, whereas the calculator methods use relative Y = 1. L*a*b* is correct under either, because it is computed against the adopted white from the same path. No spectral values are clipped: negatives (noise/fluorescence) and values > 1 (emissive/scene) are preserved as measured.

Authoritative source

The algorithms and weighting tables used here come from the ICC colorimetry-data registry — the authoritative source (380–780 nm @ 10 nm; tables computed for ICC by Tanzima Habib, NTNU).
